# 项目功能设计

## 项目定位

MindAgent 是一个面向心理健康问答与智能体交互场景的后端项目，提供认证、会话、RAG 检索、管理员能力和本地联调页面。

## 核心功能模块

### 1. 认证与权限

- 用户登录
- Token 刷新
- 基于角色的接口访问控制
- 默认角色：
  - `ADMIN`
  - `USER`

### 2. 对话能力

- 流式聊天接口
- 意图识别
- 会话消息存储
- 对话历史读取

### 3. RAG 检索

- 本地知识库加载
- Chroma 向量检索
- 管理员触发知识库重载
- 支持从仓库导入知识内容

### 4. 管理能力

- 管理员查看报告
- 管理员查看调度日志
- 管理员执行知识库相关操作

### 5. MCP 模拟能力

- Excel 写入模拟接口
- Email 发送模拟接口
- 调度结果记录

## 数据存储设计

### MySQL

主要业务表：

- `app_users`
- `chat_messages`
- `mcp_dispatch_logs`
- `psychological_reports`
- `refresh_tokens`

初始化来源：

- 首次 Docker 初始化执行 `backend/src/main/resources/db/init_schema.sql`

后续增量：

- 手工维护 `backend/src/main/resources/db/migration/`

### Redis

- 用于聊天记忆与缓存类数据

### Chroma

- 用于知识库向量检索

### 本地文件存储

- `backend/storage/knowledge`：知识库原始文本目录
- `backend/storage/repo-cache`：仓库导入后的本地缓存目录
- Chroma 仅保存向量化结果，不替代原始知识文件目录

## 本地运行设计

- 根目录 `docker-compose.yml` 统一拉起 MySQL、Redis、Chroma、nginx
- 后端服务默认运行在 `8082`
- 前端 nginx 默认运行在 `8088`

## 文档维护要求

- 功能有新增或调整时，必须同步更新本文件
- 数据库结构变更时，必须同步更新 `init_schema.sql`
