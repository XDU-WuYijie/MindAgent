# 项目进展

## 当前状态

- 项目名称已统一为 `MindAgent`
- Java 包路径已迁移为 `com.mindagent.agent`
- 默认数据库已切换为 MySQL
- 默认缓存已切换为 Redis
- Chroma 已接入本地 RAG 方案
- 聊天主链路已升级为 `IntentType + QueryType` 双层路由，继续复用现有 `POST /api/chat/stream`
- 应用层已补充 BM25 内存检索、向量检索结构化结果、RRF 融合与检索后处理，普通闲聊默认不触发 RAG
- `/api/rag/retrieve/debug` 已支持输出 `intentType / queryType / rewrittenQuery / bm25Results / vectorResults / fusedResults / selectedResults`
- `rag` SSE 事件已扩展为 JSON 调试负载，包含 `queryType`、改写后的 query 和选中 chunk 信息
- 混合检索链路已调整为 `BM25 Top30 + Vector Top30 -> RRF -> Qwen qwen3-rerank Top5 -> 前端展示 Top5`
- 用户前端已增加 RAG 调试入口，可直接查看 `queryType`、召回数量、Rerank Top5 与最终展示 chunk
- 校园内部知识库上传默认归属 `knowledge_base_3`，启动时会将误放在 `knowlegde_base_1` 的预约流程文档迁移并重建 chunk / 向量索引
- FAQ/KB 已支持按标题优先的递归切块入向量库，并为每个块写入结构化 metadata
- 管理员上传知识文档已切换为异步入库流程：原始文件落盘后写入 `kb_document`，后台解析切块写入 `kb_chunk`，数据库状态先提交为 `ACTIVE`，向量同步在事务外执行，避免长时间停留在 `PROCESSING`
- 前端已支持统一登录入口，登录后按 `ADMIN` / `USER` 角色自动分流到管理员页或用户页；首页已移除登录表单，管理员页以 MySQL 为准展示文档列表、详情和 chunk 明细；知识库上传已按两个知识库空间拆分，并在每个空间内提供 FAQ / KB 两个入口，支持同名覆盖更新
- 当前知识库文件已按 `knowledge_base_1/2/3` 预留目录，并细分为 `faq/` 与 `kb/` 子目录，旧的根目录文件已不再作为加载来源
- 知识库根路径已固定到 `backend/storage/knowledge`，避免因启动目录不同把文件写到项目根目录的 `storage/` 下
- 默认 LLM 策略已调整为主聊天与 Spring AI 均默认走 Qwen 兼容云端接口
- Spring AI 向量模型默认已切换为 `text-embedding-3-small`
- 校园心理健康知识空间已合并原来的权威科普资料入口，相关目录已删除
- 聊天记忆已升级为多会话 + MySQL 持久化 + Redis recent window 短期缓存
- 本地依赖统一通过根目录 `docker-compose.yml` 管理
- Docker 数据目录已整理到 `docker/mysql/data`、`docker/redis/data`、`docker/chroma/data`
- 后端本地运行目录已统一调整为 `backend/storage/`
- 文档与归档资料已整理到 `docs/`
- 数据库初始化脚本已落地到 `backend/src/main/resources/db/init_schema.sql`

## 已完成事项

- 完成项目从 `MindBridge` 到 `MindAgent` 的命名迁移
- 调整本地运行端口，避开已有 `campus-app`
- 配置 MySQL、Redis、Chroma 本地依赖
- 调整 nginx 反向代理到 `8082`
- 增加 MySQL 初始化脚本
- 在初始化脚本中加入默认用户数据
- 清理无必要的脚本目录
- 将知识库目录与仓库缓存目录从 `data` 迁移到 `storage`
- 新增真实会话接口：创建、列表、历史查询、删除
- 聊天上下文切换为 `session summary + recent window + current query + optional RAG`
- 增加会话摘要表、压缩日志表与 recent cache 回源/回填逻辑
- 前端静态页已支持新建/切换/删除会话与查看会话历史
- 前端已增加当前会话 Token 调试面板，可查看摘要、recent window、压缩状态与预计 prompt token
- `psychological_reports.query_text` 的实体映射已与 `init_schema.sql` 中的 `LONGTEXT` 对齐，避免 JPA schema 校验在不同平台上失败
- 知识库向量同步已改为重建式流程，重载时先清空 Chroma 集合再重新写入当前知识块
- 知识文档已增加 MySQL 持久化状态流转，支持 `PROCESSING / ACTIVE / FAILED` 和文档状态查询
- 管理端已增加知识文档列表、详情、chunk 明细和软删除接口
- 新增 QueryType 路由、BM25 检索、RRF 融合和检索后处理相关服务与单测
- 修复 `/api/rag/retrieve/debug` 在 WebFlux 下的阻塞式 `block()` 导致的 `500` 问题，并开放 `USER` 角色访问
